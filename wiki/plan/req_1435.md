# 执行计划：数据编辑面板 + 消息复制按钮

## 需求概要

1. **数据编辑面板**：独立遮罩面板（类似 DebugPanel），用户可编辑 module、contextHistory、messages 的内容（非 JSON 原始格式），实时生效
2. **消息复制按钮**：每个消息气泡下方增加复制按钮，点击复制正文（保留 Markdown 格式），按钮文字短暂变"已复制"

### 已对齐的关键决策

| 维度 | 决定 |
|---|---|
| 数据范围 | module + contextHistory + messages |
| 面板形态 | 独立遮罩面板（类似 DebugPanel，右侧滑出） |
| 可编辑字段 | 仅 `content` |
| 生效方式 | 实时生效（onChange 直接写 dataStore） |
| 编辑控件 | 结构化表单（独立 textarea） |
| 入口 | TopBar 按钮，始终可见 |
| system 消息 | 过滤不显示 |
| contextHistory | 每个条目独立 textarea |
| 消息排序 | 时间正序（旧→新） |
| 复制内容 | 仅正文 content，保留 Markdown |
| 反馈方式 | 按钮文字短暂变"已复制" |
| 按钮位置 | 气泡卡片外部下方 |

## 风险与约束声明

- **contextHistory 分隔符冲突**：条目内如果包含 `\n\n`，拆分时会断裂。当前遵循 `appendContextHistory` 的 `\n\n` 分隔约定，属已知限制
- **实时编辑风险**：编辑中的消息如果同时被 AI 生成过程修改，可能产生竞争。实际使用中两者通常不会同时进行，风险较低
- **大消息量性能**：消息数量极大时（100+），每个 textarea onChange 触发 `setData` 可能有延迟

## 测试验证方案

1. **复制按钮**：点击复制 → 验证剪贴板内容（保留 Markdown） → 按钮变"✓ 已复制" → 1.5s 恢复
2. **数据编辑面板**：打开面板验证三个区域数据正确 → 编辑 module 验证生效 → 编辑消息 content 验证实时更新 → 删除 contextHistory 条目验证 → system 消息过滤 → 空数据场景
3. **整合验证**：`npx tsc --noEmit` 零错误 → 面板开关不破坏对话流程 → 两个面板同时打开不冲突
