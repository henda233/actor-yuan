# 需求1435：数据编辑面板 + 消息复制按钮

## 需求描述

1. **数据编辑页面**：实现一个页面，用户可以直接编辑当前的数据（上下文、对话等等），不要直接显示JSON内容，需要解析JSON的内容，便于用户修改调整。
2. **消息复制**：在每个消息气泡的下面增加功能，点击后复制当前气泡的内容。

## 需求细节（对齐后确认）

### 数据编辑面板

1. **数据范围**：module + contextHistory + messages（不含 billing）
2. **页面形态**：独立遮罩面板（类似 DebugPanel，右侧滑出）
3. **可编辑字段**：仅 `content`（消息正文）
4. **生效方式**：实时生效（onChange 直接写 dataStore）
5. **编辑控件**：结构化表单（每个字段独立的 textarea）
6. **入口**：TopBar 按钮，始终可见
7. **过滤规则**：不显示 system 消息
8. **contextHistory**：每个条目独立 textarea，可单独修改或删除
9. **消息排序**：时间正序（旧→新）

### 消息复制按钮

1. **复制内容**：仅消息正文 `content`
2. **复制格式**：保留 Markdown 格式
3. **反馈方式**：按钮文字短暂变为"✓ 已复制"（1.5s）
4. **按钮位置**：气泡卡片外部下方
