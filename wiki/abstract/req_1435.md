# 摘要：需求1435-数据编辑面板与消息复制

## 核心结论与关键信息

- 新增 **DataEditorPanel**：独立遮罩面板（z-index: 99），编辑 module + contextHistory + messages 的 content 字段，实时写 dataStore
- 新增 **复制按钮**：每个消息气泡下方，`navigator.clipboard.writeText` 复制正文（保留 Markdown），按钮文字 1.5s "✓ 已复制" 反馈
- `contextHistory` 以单个 textarea 整体编辑，dataStore 新增 `setContextHistory` 方法直接读写整个字符串
- 面板宽度 80vw，面板入口：TopBar 新增按钮，始终可见（非 Debug 限制）
- system 消息在编辑器中过滤不显示
- 消息按时间正序排列，仅 `content` 字段可编辑

## 内容概述

需求包含两个独立功能：
1. **数据编辑面板**：结构化为 module（textarea）/ contextHistory（单个 textarea 整体编辑）/ messages（每条独立 textarea）三个编辑区，实时编辑
2. **复制按钮**：MessageBubble 外包裹 wrapper，下方新增透明小按钮

复用 DebugPanel 的 overlay + 滑动面板模式。dataStore 需扩展 contextHistory 操作方法。

## 依赖与影响链

- **上游依赖**：`src/types/storage.ts`、`src/services/dataStore.tsx`、`src/components/DebugPanel.tsx`（overlay 模式参考）
- **下游被依赖**：`src/components/DataEditorPanel.tsx`（新建）、`src/App.tsx`、`src/components/TopBar.tsx`
- **变更扩散评估**：中（dataStore 新增方法不影响现有消费者；MessageBubble 外层包裹需注意 flex 布局兼容）
