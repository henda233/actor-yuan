# 摘要：需求4 —— UI 设计与交互模式

## 核心结论与关键信息

- **主色调**：白、蓝、黑，简洁风格
- **两种交互模式**（与需求2对应）：
  1. **直接对话 UI**：常规 AI 对话界面，用户输入玩家操作，AI 输出游戏情节
  2. **插入对话 UI**：用户在游戏情节中**右键某个位置**，弹出输入框，输入判定结果等内容
- 两种模式服务于需求2定义的两阶段交互流程
- **计费与用量展示**：对话界面角落常驻显示当前会话的 token 用量和预估费用（来自 req6）
- **设置入口**：需提供计费价格配置入口（各模型输入/输出每千 token 价格）
- **模组管理入口**：需提供模组导入入口（文件上传 + 文本框粘贴两种方式），以及查看/编辑已导入模组的界面

## 内容概述

从 UI 角度定义了用户与 AI 主持人的交互方式。直接对话模式是常规的聊天界面；插入对话模式是 TRPG 特有的交互——用户需要在情节的特定位置（句子/段落前）插入判定结果，因此 UI 需要支持精确的文本位置定位和右键菜单触发。此外，计费信息和模组管理作为辅助功能集成在界面中。
