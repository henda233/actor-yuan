# ActorYuan 部署文档

> 适用环境：Ubuntu 22.04 + Nginx（纯静态文件部署）

## 一、环境说明

| 项目 | 说明 |
|---|---|
| 服务器系统 | Ubuntu 22.04（裸机，新装系统） |
| Web 服务 | Nginx |
| 部署方式 | IP + 端口访问，无域名，无 HTTPS |
| 部署路径 | `/var/www/actor-yuan/` |
| 构建产物 | `dist/`（本地执行 `npm run build` 生成） |

ActorYuan 是纯前端 SPA 应用，`dist/` 目录为静态文件（HTML、JS、CSS），无后端服务，无数据库。

---

## 二、安装 Nginx

```bash
# 更新包索引
sudo apt update

# 安装 Nginx
sudo apt install nginx -y

# 确认安装成功
nginx -v
```

安装完成后 Nginx 自动启动。验证服务状态：

```bash
systemctl status nginx
```

输出中应包含 `active (running)`。

---

## 三、配置防火墙

Ubuntu 22.04 默认使用 `ufw`。如果开启了防火墙，需要放行 HTTP 端口：

```bash
# 查看防火墙状态
sudo ufw status

# 如果防火墙处于 active 状态，放行端口
sudo ufw allow 80/tcp          # HTTP

# 如果你选择其他端口（如 8080），改为对应的端口号
sudo ufw allow 8080/tcp
```

> **注意**：如果你使用的是云服务器（阿里云、腾讯云、AWS 等），还需要在云控制台的**安全组**中放行对应端口。

## 四、部署文件

### 4.1 创建目录

```bash
sudo mkdir -p /var/www/actor-yuan
```

### 4.2 上传 dist 文件

将本地 `dist/` 目录下的**全部内容**上传至服务器 `/var/www/actor-yuan/`。

上传后目录结构应为：

```
/var/www/actor-yuan/
├── index.html
├── assets/
│   ├── index-xxxxx.js
│   ├── index-xxxxx.css
│   └── ...
└── ...
```

> 文件传输方式由你自行决定（scp、rsync、SFTP 等）。

### 4.3 设置权限

```bash
# 确保 Nginx 能够读取文件
sudo chown -R www-data:www-data /var/www/actor-yuan/
sudo chmod -R 755 /var/www/actor-yuan/
```

## 五、配置 Nginx

### 5.1 创建站点配置

Nginx 的站点配置文件放在 `/etc/nginx/sites-available/`，通过符号链接到 `/etc/nginx/sites-enabled/` 启用。

创建配置文件：

```bash
sudo vim /etc/nginx/sites-available/actor-yuan
```

填入以下内容：

```nginx
server {
    listen 80;
    server_name _;                          # 不绑定域名，响应所有 IP 请求

    root /var/www/actor-yuan;
    index index.html;

    # Gzip 压缩 —— 减少传输体积
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml text/javascript;
    gzip_min_length 256;
    gzip_vary on;

    # SPA 路由：所有路径 fallback 到 index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源强缓存（文件名包含哈希，内容变更时自动失效）
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

**关键配置解释**：

| 配置项 | 说明 |
|---|---|
| `listen 80` | 监听 HTTP 80 端口 |
| `server_name _` | 不匹配任何特定域名，所有到此端口的请求都由本 server 块处理 |
| `try_files $uri $uri/ /index.html` | 先尝试匹配实际文件，找不到时返回 `index.html`（由 React Router 接管路由） |
| `gzip` 系列 | 对文本类文件开启压缩，减少传输体积 |
| `/assets/` 缓存 | Vite 构建的静态资源文件名含哈希，可安全设置长期缓存 |

### 5.2 启用站点

```bash
# 移除默认站点（可选，避免端口冲突）
sudo rm /etc/nginx/sites-enabled/default

# 创建符号链接启用 actor-yuan 站点
sudo ln -s /etc/nginx/sites-available/actor-yuan /etc/nginx/sites-enabled/

# 检查配置语法
sudo nginx -t
```

`nginx -t` 输出 `syntax is ok` 和 `test is successful` 即配置正确。

### 5.3 重载 Nginx

```bash
sudo systemctl reload nginx
```

## 六、验证部署

在浏览器中访问 `http://<服务器IP>:80`，应看到 ActorYuan 的欢迎页面。

验证清单：

- [ ] 浏览器打开 `http://<服务器IP>` 能正常加载页面
- [ ] 页面上的设置面板能正常操作
- [ ] 刷新页面不出现 404（验证 SPA 路由 fallback 生效）
- [ ] 如果使用非 80 端口，确认端口在防火墙/安全组已放行

---

## 七、常用管理命令

```bash
# 启动 Nginx
sudo systemctl start nginx

# 停止 Nginx
sudo systemctl stop nginx

# 重载配置（不中断服务）
sudo systemctl reload nginx

# 重启 Nginx
sudo systemctl restart nginx

# 查看错误日志
sudo tail -f /var/log/nginx/error.log

# 查看访问日志
sudo tail -f /var/log/nginx/access.log

# 开机自启
sudo systemctl enable nginx
```

---

## 八、更换端口（可选）

如需使用其他端口（如 8080），修改配置文件中的 `listen` 指令：

```nginx
server {
    listen 8080;
    # ... 其余配置不变
}
```

修改后记得：
1. `sudo nginx -t` 检查语法
2. `sudo systemctl reload nginx` 重载
3. 放行对应端口：`sudo ufw allow 8080/tcp`

---

## 九、故障排查

| 现象 | 检查项 |
|---|---|
| 浏览器无法访问 | `systemctl status nginx` 确认 Nginx 运行中；`sudo ufw status` 确认端口已放行；云服务器检查安全组规则 |
| 刷新页面出现 404 | 检查 `try_files $uri $uri/ /index.html;` 是否配置正确 |
| 页面白屏 / JS 报错 | 检查 `/var/www/actor-yuan/` 下文件是否完整（`index.html` + `assets/` 目录） |
| 修改配置不生效 | 确认已执行 `sudo systemctl reload nginx` |
| 权限错误 | `sudo chown -R www-data:www-data /var/www/actor-yuan/` |
