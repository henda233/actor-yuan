3. 现在的功能是AI给出情节内容后，默认为草稿，直接就可以编辑。需要改为：AI给出情节后，对应的气泡的下面有几个按钮：重新生成（情节，推演方案不重新生成）、放弃草稿、编辑草稿、确认草稿。点击`编辑草稿`后，情节会变为可编辑状态，按钮也会变为：保存，取消。当点击保存后，该插入信息的情节会自动返回给AI进行重写；如果点击取消，则回到上面几个按钮的状态。只有用户点击`确认草稿`后，将该草稿视为正式的游戏情节。注意，当前项目没有实现编辑草稿，确认草稿后将情节返回给AI然后重写的功能。特别地，插入到草稿中的前面的内容视为正式情节（即【】之前的内容），该正式情节直接加入到上下文中，AI重写后的内容是不包含【】之前的内容的（这个由提示词控制，不是项目的职责），举个例子：

第一次草稿：
```
林恩没有看勋爵。勋爵的话还在耳边嗡嗡作响——“灰崖哨站”、“全员无幸存”、“预备民兵”——但这些信息已经被他暂时归档到脑内笔记簿的“待处理”一栏。此刻他的注意力正穿过前院里交头接耳的镇民，锁定在人群后排那个磨坊户男人的脸上。
可惜，林恩没有找到那个奇怪磨坊户男人，可能是人群太密集了，让林恩目不暇接....后面省略
```
插入判定内容（计算内容）后的草稿：
```
林恩没有看勋爵。勋爵的话还在耳边嗡嗡作响——“灰崖哨站”、“全员无幸存”、“预备民兵”——但这些信息已经被他暂时归档到脑内笔记簿的“待处理”一栏。此刻他的注意力正穿过前院里交头接耳的镇民，锁定在人群后排那个磨坊户男人的脸上。【判定结果：林恩的侦查判定成功。】...这里后面的内容省略。
```

此时【】前面的内容“林恩没有看...磨坊户男人的脸上”视为正式情节（因为它们都没有被插入内容，说明是被外部系统或者用户承认了），然后将上面的内容返回给AI后，AI给出修改后的内容：
```
磨坊户站在靠近院门口的位置，旁边是他那个缠着绷带的妻子。男人在听到“灰崖哨站”四个字时没有任何反应——没有惊讶，没有恐惧，甚至连眉头都没有皱一下。...后面省略
```
可以看到上面的内容中，【】包含它前面的内容AI不会再输出，注意这部分的截取是由AI负责的，不是当前项目的职责。
